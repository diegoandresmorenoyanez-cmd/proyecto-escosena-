Requisitos Funcionales (Resumen):
RF01: El sistema debe permitir el registro de nuevos usuarios.
RF02: El sistema debe permitir el inicio y cierre de sesión.
RF03: El administrador debe poder publicar nuevas noticias.
RF04: El administrador debe poder editar noticias publicadas.
RF05: El administrador debe poder eliminar noticias.
RF06: El sistema debe clasificar las noticias por categorías (deportes, política, tecnología, entretenimiento, etc.).
RF07: El sistema debe permitir buscar noticias por título o palabras clave.
RF08: El sistema debe mostrar las noticias más recientes en la página principal.
RF09: Los usuarios deben poder comentar en las noticias (si tienen sesión iniciada).
RF10: El sistema debe permitir calificar las noticias mediante reacciones (Me gusta).
RF11: El sistema debe permitir compartir noticias en redes sociales.
RF12: El sistema debe mostrar noticias relacionadas al final de cada publicación.
RF13: El sistema debe permitir guardar noticias en favoritos.
RF14: El administrador debe poder gestionar usuarios (crear, editar, bloquear o eliminar).
RF15: El sistema debe enviar notificaciones a los usuarios sobre noticias destacadas o de última hora.
Requisitos No Funcionales:
RNF01: El sistema debe cargar la página principal en un tiempo menor a 3 segundos con una conexión estándar.
RNF02: El portal debe estar disponible al menos el 99.5% del tiempo.
RNF03: La información de los usuarios debe almacenarse de forma segura mediante contraseñas cifradas.
RNF04: El sistema debe ser compatible con los navegadores Google Chrome, Mozilla Firefox, Microsoft Edge y Safari.
RNF05: El portal debe ser adaptable a dispositivos móviles, tabletas y computadores (Diseño Responsive).
RNF06: El sistema debe soportar al menos 1.000 usuarios simultáneos sin degradar significativamente su rendimiento.
RNF07: La interfaz debe ser intuitiva y fácil de usar para usuarios de cualquier nivel.
RNF08: El sistema debe realizar copias de seguridad automáticas diariamente.
RNF09: El portal debe cumplir con estándares básicos de accesibilidad (como WCAG).
RNF10: Todas las comunicaciones entre el cliente y el servidor deben realizarse mediante HTTPS.
RNF11: El sistema debe registrar los errores y eventos importantes en archivos de auditoría.
RNF12: El código del sistema debe estar documentado para facilitar su mantenimiento.
RNF13: El sistema debe permitir futuras ampliaciones sin afectar el funcionamiento actual (escalabilidad).
RNF14: El portal debe mostrar correctamente imágenes y contenido multimedia sin afectar el rendimiento.
RNF15: El tiempo de recuperación del sistema ante una falla crítica no debe superar 30 minutos.
